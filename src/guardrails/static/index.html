About this model

A binary prompt-safety classifier. Returns one verdict per prompt: safe, or an attempted prompt-injection / jailbreak. Drop it in front of your LLM calls as a cheap first-pass filter; your app decides what to do with the verdict.

Base model

Fine-tuned from answerdotai/ModernBERT-base for binary classification. ~570 MB on disk. SDPA attention; runs on MPS, CUDA or CPU.

Training data

Primary corpus

  • leolee99/PIGuard - 76.7k labelled prompts pooled from 20 public sources spanning benign requests, prompt injections and jailbreaks.

Optional hard-negative augmentation

  • leolee99/NotInject (splits NotInject_one + NotInject_two) - 226 benign prompts that contain attack vocabulary. On by default.
  • allenai/wildjailbreak (adversarial_benign subset) - ~78k contrastive benign prompts. Off by default (gated dataset).

Held out for OOD evaluation (not used in training)

  • NotInject_three split
  • XSTest
  • deepset/prompt-injections
  • allenai/wildjailbreak eval split

Goals

  • Catch prompt-injection / jailbreak attempts before they reach the upstream model
  • Low latency so it is cheap to run on every request
  • Verdict-only: no refusal text, no rewrites, no policy opinions
  • Portable: same weights usable in-process, over HTTP, or via ONNX

Known blind spots

  • Role-play framings without trigger words
  • Non-English attacks (training set is English-heavy)
  • Multi-turn attacks (stateless; scores each prompt independently)
  • Images (text only; OCR upstream first)
